Стиль кода HTML Academy

HTML

Синтаксис

  • Для отступов у вложенных элементов используйте два пробела. Ширина табов может быть различной в разных редакторах и системах, в то время как пробел везде одной ширины (TODO: ссылка на файл .editorconfig).
  • Теги и их атрибуты пишите строчными буквами. Для значений атрибутов всегда используйте двойные кавычки.
  • Не используйте необязательный закрывающий слеш у одиночных тегов ( <img>, <br> и другие).
  • Не пропускайте необязательные закрывающие теги (например, </li> или </body>).

Валидный код

Документ должен проходить проверку на валидность. Для проверки используйте современный валидатор.

HTML5 doctype

В начале страницы обязательно нужно указывать актуальный doctype, чтобы браузер отображал её в режиме соответсвия стандартам. Это гарантирует, что страница будет выглядеть единообразно во всех современных браузерах.

Семантическая разметка

Используйте в разметке семантические теги HTML.

Параграфы текста помещайте в тег <p>. Не создавайте для отступов между текстовыми блоками много тегов br.

Кодировка символов

Всегда явно указывайте кодировку символов на странице, чтобы обеспечить корректное отображение текста. Кодировка utf-8 предпочительна.

Минималистичность разметки

В разметке нужно использовать минимальное возможное количество элементов. Не должно быть лишних оберток и блоков, которые используются для оформления и могут быть заменены на псевдоэлементы.

Количество CSS-файлов

Минимизируйте количество CSS-файлов, подключенных к странице. Меньшее количество запросов к серверу улучшит быстродействие страницы.

Все стили страниц должны находится в едином файле. Исключением могут являться файлы со сбросом стилей normalize.css или reset.css. Такой файл может быть подключен отдельно, перед основным стилевым файлом.

Подключение CSS-файлов

Стилевые файлы с помощью <link> нужно подключать внутри <head>. При этом атрибут type для тега <link> указывать не нужно, так его значение text/css указывается для CSS-файлов по умолчанию.

Подключение JavaScript-файлов

Скрипты нужно подключать в самом низу страницы, чтобы при её загрузке скрипты не блокировали содержимое страницы.

При подключении JavaScript-файлов в теге <script> не обязательно указывать атрибут type, так его значение text/javascript указывается для JS-файлов по умолчанию.

Протокол ресурса

При запросе файла через полный URL часть с протоколом (http:, https:) из адреса можно исключить.

Это сделает URL относительным и немного уменьшит размер файлов.

Порядок следования атрибутов

Классы HTML-элементов используются для описания компонентов и прежде всего они могут меняться. Поэтому атрибут класса у элементов следует писать первым.

Логические атрибуты

Для логических атрибутов элементов (например, checked, disabled, required) не обязательно указывать значение. Достаточно наличия самого атрибута.

Подписи полей ввода

Для улучшения взаимодействия пользователя с элементами форм, при нажатии на подпись поля, оно должно активироваться. Для этого связывайте элемент формы с его описанием, используя тег <label> и идентификаторы.

Размеры картинок

Картинкам <img> нужно задать действительные размеры без указания единиц размерности. Это не будет заставлять браузер перерисовывать резметку в процессе загрузки и отображения картинки.

Атрибут языка

Для элемента <html> в атрибуте lang нужно указывать соответствующий язык документа. Это помогает инструментам синтеза речи определить какое использовать произношение или системам перевода — какие использовать правила.

CSS

Синтаксис

  • После значения свойства обязательно ставьте точку с запятой.
  • Для отступов внутри правил используйте два пробела. Ширина табов может быть различной в разных редакторах и системах, в то время как пробел везде одной ширины (TODO: ссылка на файл .editorconfig и .csscomb.json).
  • Шестнадцатиричное значение цвета пишите полностью из всех шести символов, не сокращайте. Для записи используйте строчные буквы. Например, #f5f5f5.
  • Названия тегов и свойств в правилах пишите строчными буквами.
  • Не добавляйте начальный ноль для значений (например, .5 вместо 0.5).
  • Во всех случая в CSS используйте двойные кавычки, а не одинарные. Не опускайте кавычки в необязательных случаях.
  • После двоеточия в правилах ставьте один пробел (top: 10px;). А перед двоеточием пробел не нужен.
  • Не оставляйте пробелов после запятых внтури значений rgb(), rgba(), hsl(), hsla() или rect(). Это помогает различать различные цветовые значения (запятая без пробела) от нескольких значений одного свойства (запятая с пробелом).
  • До и после комбинатора между селекторами (например, p > a) ставьте один пробел.
  • Каждое объявление в правиле пишите на новой строке.
  • Перед открывающейся фигурной скобкой ставьте один пробел. После скобки начинайте писать с новой строки: 
    .selector {
  color: #f5f5f5;
}
  • После запятой между селекторами ставьте один пробел, до запятой пробел не нужен: .selector-1, .selector-2 {.
  • Закрывающую фигурную скобку пишите на новой строке и без отступа.
  • Не пишите единицы измерения, там где в них нет необходимости. Например, border: 0.

Нормализация базовых стилей

Используйте normalize.css или reset.css (но не оба сразу). Подключайте один из них отдельным файлом перед основным файлом стилей.

Не используйте @import

По сравнению с тегом <link> правило @import медленнее и вызывает дополнительные запросы к серверу. Не используйте @import.

Селекторы

Выбирайте селектором в точности те элементы, что нужно. Не полагайтесь на совпадение или случайность.

Пишите такие селекторы, чтобы их можно было переиспользовать в дальнейшем. Не создавайте без необходимости цепочки селекторов или селекторы с излишней вложенностью — это увеличивает специфичность правил и уменьшает возможность их переиспользования.

Также длинные цепочки вложенных селекторов усложняют код и его поддержку. Хорошим подходом считается использовать вложенность до 2 или 3 уровня. Псевдоэлементы или псевдоклассы не увеличивают уровень вложенности.

Не используйте id для стилизации. Вместо идентификаторов для задания стилей используйте селекторы по классам или тегам.

Имена классов

  • Записывайте имена классов строчными буквами и используйте дефис (но не знаки нижнего подчеркивания или camelCase). Дефисы служат разделителями во взаимосвязанных классах (например, .btn и .btn-danger).
  • Избегайте чрезмерные и произвольные сокращения. Например, имя .btn подойдет для button, а имя .s не означает ничего.
  • Придерживайтесь настолько коротких и ёмких имен классов, насколько это возможно.
  • Используйте осмысленные имена: давайте структурные или содержательные имена вместо презентационных.
  • Для написания классов используйте английские слова и термины. Не пишите названия классов и атрибутов транслитом.

Используйте !important мудро

В большинстве случаев необходимость использования !important в CSS возникает из-за неправильного обращения к элементам страницы через каскад. Однако, в редких случаях, без использования !important не обойтись. Используйте !important мудро.

Подробнее в статье: http://meiert.com/en/blog/20150310/important/.

Сокращенная запись

Старайтесь не использовать сокращенные объявления в тех случаях, когда это может косвенно переопределить другие значения. Наиболее часто злоупотребляют сокращением следующих свойств:

  • padding
  • margin
  • font
  • background
  • border
  • border-radius

Чрезмерное использование сокращенной записи свойств часто приводит к грязному коду с ненужными переопределения и непреднамеренными побочными эффектами.

Варианты шрифта

Указывайте альтернативные варианты шрифта и тип семейства в конце перечисления font-family.

Альтернативный веб-безопасный шрифт и тип семейства необходимо указывать для того, чтобы в случае отсутствия основного шрифта, изменения внешнего вида шрифтов на странице были минимальны. Альтернативный шрифт должен быть такого же типа, что и основной.

Порядок шрифтов следующий:

  1. основной шрифт;
  2. веб-безопасный;
  3. тип шрифта.

Список веб-безопасных шрифтов можно посмотреть здесь: http://www.cssfontstack.com/.

Указывайте vertical-align

Явно указано подходящее vertical-align для потоковых блоков с display: inline-block.

По умолчанию элементы inline-block имеют выравнивание baseline, которое может приводить к нежелательному поведению элементов, например, когда внутри них разное количество строк.

Поэтому нужно явно и осознанно задавать выравнивание таким элементам.

Например, если вы хотите, чтобы элементы вели себя как текст, стоит явно указать значение baseline, такое указание позволит другим понять, что вы знаете об особенностях поведения элементов с этим выравниванием и задали его намеренно.

Свойства с префиксами

Когда вы используете свойства с вендорными префиксами, оставляйте отступы для каждого свойства так, чтобы значения объявлений выстраивались в вертикальную линию. Это упрощает многострочное редактирование.

В списке свойств с префиксами указывайте вариант без префикса последним.

Медиа-выражения

Помещайте медиа-выражения рядом с соответствующими наборами правил. Не объединяйте их в отдельную таблицу стилей. Не помещайте их в конце файла. Их должно быть просто найти.
